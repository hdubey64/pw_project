Execution Context :-

Execution context is a runtime environment in JS, which executes JS code. It stores variables, functions, objects and other runtime details. Each function call or code block has its own execution context.

Variable Object: -

Variable Object is creating for each function or code block in execution context. Variable object stores variables and their value.

Scope Chain: -

Each function or code block holds its own reference to the outer scope or lexical scope. The scope chain allows access to them.

this Value: -

The “this” keyword refers to the context in which a function is executed. The value of this is determined dynamically each time function is called, based on how the function is invoked. It allows access to the object on which a method is called or the context in which a function is executed

Outer Environment: -

The execution context consists of the outer environment (the parent scope), which may be the global scope or the scope of an enclosing function.

Type of Execution Context: -

1. Global Execution context: -

The global execution context is the starting point of any programming. It is a bigger execution context than any other. The global execution context is created when JS code execution starts. It stores functions with global scope and globally accessible.

2. Function Execution context: -

The function execution context is responsible for the function code execution. It stores the local variables of the function, the function parameters and the internal function of the function. Calling a function creates a function execution context.
